## 风险与回退

### 风险等级：低-中

**潜在问题**:

1. **删除代码导致意外破坏**
   - **影响**：删除了仍在使用的代码
   - **缓解**：使用 IDE 查找引用，谨慎删除
   - **回退**：保留删除列表，撤销提交

2. **日志级别调整过度**
   - **影响**：重要信息丢失，调试困难
   - **缓解**：保留关键操作的 INFO 日志
   - **回退**：恢复部分日志级别

3. **Legacy 路径标注不清**
   - **影响**：用户不知如何回退到旧行为
   - **缓解**：清晰文档化配置选项
   - **回退**：补充文档说明

4. **RC 构件问题**
   - **影响**：发布版本不稳定
   - **缓解**：充分手动测试
   - **回退**：发布 RC2 修复问题

**回退方案**:
- 所有删除操作记录在清单中，可逐条恢复
- 保留 git 分支，可整体回退
- 发布前充分测试，确保稳定性
